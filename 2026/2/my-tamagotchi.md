## Описание проекта 
Название проекта: Консольный Тамагочи
Разработка интерактивного консольного приложения, моделирующего уход за виртуальным питомцем. Проект фокусируется на управлении ресурсами, простой экономике и комфортном геймплее.
Ключевые принципы работы:
Изменение всех показателей происходит только в результате действий игрока. 
Критические значения (0) не приводят к завершению игры. Они временно ограничивают список доступных команд, мотивируя игрока восстанавливать нужный ресурс.
Экономика построена на цикле: Мини-игры → заработок монет → Кормление/Отдых → восстановление статов → Прогулка/Игры.
Прогресс полностью привязан к имени игрока, что позволяет нескольким пользователям запускать приложение на одном компьютере с независимыми сохранениями.


## Функциональность 
1. Инициализация и регистрация
Ввод имени игрока для создания индивидуального профиля.
Создание питомца с заданием уникального имени (стартовые показатели фиксированы и сбалансированы под единственный вид).
Автоматическая загрузка сохранённого прогресса при повторном входе под тем же именем игрока.
2. Система показателей
Три основных параметра в диапазоне 0–100:
Радость (эмоциональное состояние)
Энергия (ресурс для активных действий)
Сытость (физическое состояние)
Баланс монет: внутренняя валюта без верхнего лимита, не может стать отрицательной.
Визуализация: числовые значения и текстовые прогресс-бары в консоли.
3. Взаимодействие с питомцем
Прогулка: увеличивает Радость, расходует Энергию и Сытость, приносит фиксированный доход в монетах.
Кормление: восстанавливает Сытость за монеты.
Отдых: восстанавливает Энергию без затрат.
Проверка условий: действия доступны только при достаточных значениях показателей и наличии монет.
4. Мини-игры и экономика
Реализация 1–2 консольных мини-игр.
Запуск игры расходует Энергию питомца.
Мини-игры выступают основным источником заработка валюты для последующего ухода за питомцем.
5. Сохранение прогресса
Отдельный файл сохранения для каждого игрока.
Автосохранение данных после каждого завершённого действия.
Загрузка актуальных показателей при старте приложения.
6. Интерфейс и управление
Консольное меню с навигацией по цифрам.
Валидация ввода: обработка пустых строк, букв, чисел вне допустимого диапазона.
Отсутствие механики «смерти»: при достижении показателя 0 игра не завершается, а временно блокирует связанные действия до восстановления ресурса.
